# Декомпозиция: локальный переключатель раскладки (macOS, Swift)

Подход: снизу вверх, риск снимаем первым. Движок детекции разрабатывается изолированно и доводится на корпусе до включения в пайплайн. Надёжность (event tap, fail-open) закладывается в фундамент, а не «потом». Оценки ориентировочные, для одного разработчика: **S** ≤ 1 дн, **M** 2–4 дн, **L** 1–2 нед.

## 1. Архитектура: модули и зоны ответственности

1. **InputCapture** — CGEventTap, run loop, жизненный цикл tap (перевзведение при отключении), маршрутизация событий, неблокирующий колбэк с уходом тяжёлой логики в async-очередь. Владеет REL-1, REL-2, частью REL-3/REL-4. Фундамент и главный риск.
2. **KeystrokeBuffer** — буфер текущего слова по контексту, границы слова, обработка backspace/правок, сброс по навигации/клику/смене фокуса, немедленная очистка (приватность). Эфемерный.
3. **ContextProvider (Accessibility)** — AX-наблюдатель активного приложения и фокусного поля, роль поля (AXSecureTextField, url/email/search-эвристики), позиция курсора для индикатора, чтение/замена текста через AXValue. Питает проактивную установку, детект защищённых полей, индикатор.
4. **DetectionEngine** — ядро: по слову и контексту решает (валидно в текущей раскладке? валидно в другой? confidence). Частотные словари / n-gram, порог, политика «при сомнении не трогать». Чистый, детерминированный, тестируемый.
5. **LayoutController** — чтение/установка input source (TIS), таблицы keycode↔символ RU/EN, исполнение конвертации (AXValue, fallback на синтетический backspace), атомарность операции с отменой при гонке. Единый источник состояния раскладки + память по приложению/полю.
6. **ActionCoordinator** — стейт-машина-оркестратор: граница слова → движок → исполнение; хоткеи петли исправления, hold-to-suppress, глобальный тумблер, shadow-mode. Здесь живёт политика.
7. **Store** — словарь исключений, белый список «всегда латиница», память раскладки, выученные реверты; локальная атомарная запись, import/export.
8. **HotkeyManager** — глобальные хоткеи, детект одиночного/двойного модификатора без поедания обычного Shift, переназначение.
9. **UI** — menu bar (NSStatusItem, LSUIElement), caret-оверлей (borderless non-activating panel), окно настроек (SwiftUI), панель shadow-обзора, обратная связь (звук/вспышка).
10. **Lifecycle/Supervision** — LaunchAgent KeepAlive, single-instance, онбординг разрешений и детект их отзыва, наблюдатели сна/смены монитора/смены пользователя, восстановление состояния.
11. **TestHarness** — локальный раннер корпуса по DetectionEngine, метрики accuracy/false-positive, бенч латентности. Без сети.

## 2. Граф зависимостей модулей

InputCapture → KeystrokeBuffer → ActionCoordinator.
ActionCoordinator ← DetectionEngine, LayoutController, ContextProvider, Store, HotkeyManager.
UI ← состояние ActionCoordinator/Store. Lifecycle оборачивает всё. TestHarness зависит только от DetectionEngine.

## 3. Модель данных (всё локально, атомарная запись, без сети)

- Настройки: тумблеры, порог уверенности, хоткеи, обратная связь — JSON/plist.
- Словарь исключений (не конвертировать) — множество.
- Белый список «всегда латиница» — множество.
- Память раскладки: bundleID + роль поля → раскладка.
- Выученные реверты — множество со счётчиками.
- Частотные словари RU/EN — в комплекте, read-only, в память.
- Shadow-лог — кольцевой буфер последних N, эфемерный/локальный.

## 4. Фаза 0 — Технические спайки (снять неизвестность до фиксации архитектуры)

- **S0.1** Прототип CGEventTap: установка, run loop, лог событий, принудительное отключение tap и авто-перевзведение, замер латентности колбэка (M). Самый высокий риск → первым.
- **S0.2** Чтение/замена текста через AXValue на Safari, Chrome, VS Code (Electron), Slack, нативных полях; матрица поддержки, выбор стратегии fallback (M).
- **S0.3** Переключение input source (TIS) + таблицы keycode↔символ RU/EN, round-trip тест (M).
- **S0.4** Детект защищённого поля и IME-композиции (dead keys, диктовка, accent-меню) (S).

Exit: выбраны механизмы конвертации и переключения; подтверждены бюджет латентности и работоспособность авто-перевзведения; известно, где AXValue не работает и нужен backspace-fallback.

## 5. Фазы и эпики

### EPIC 1 — DetectionEngine (офлайн, тестируемый)
- **E1.1** Формат и загрузка частотных словарей RU/EN в память (M).
- **E1.2** Скоринг: валидность слова в раскладке + n-gram/частоты, выдача confidence (L).
- **E1.3** Порог уверенности и политика «при сомнении — бездействие» (S).
- **E1.4** Контекстный сигнал: язык окружающего текста как вход скоринга (M).
- **E1.5** Тестовый корпус + регрессионный раннер, метрики accuracy/false-positive, бенч латентности (M).
- Exit: на корпусе false-positive ≤ 0.5%, детерминизм, латентность анализа существенно ниже бюджета.

### EPIC 2 — Input Pipeline (MVP автоконвертации)
- **E2.1** InputCapture поверх спайка: маршрутизация, неблокирующий колбэк, async-очередь (M).
- **E2.2** KeystrokeBuffer: слово/границы, backspace/правки, сброс по навигации/клику/фокусу, очистка (M).
- **E2.3** LayoutController: чтение/установка раскладки, маппинг символов, конвертация (AXValue + fallback), атомарность с отменой при гонке (L).
- **E2.4** ActionCoordinator (стейт-машина): граница слова → движок → исполнение; fail-open на всех путях (M).
- **E2.5** REL-1 авто-перевзведение tap в проде; REL-3 сквозной fail-open (S).
- Exit: «ghbdtn → привет» стабильно; tap самовосстанавливается; ввод не тормозит и не теряется.

### EPIC 3 — Ручное управление и петля исправления
- **E3.1** HotkeyManager: глобальные хоткеи, детект одиночного/двойного модификатора без поедания обычного Shift (M).
- **E3.2** Конвертация последнего слова/строки/выделения; повторное нажатие — откат/цикл (M).
- **E3.3** Hold-to-suppress; глобальный тумблер авто; ручное переключение раскладки (S).
- **E3.4** Отмена автоконвертации без конфликта с нативным Cmd+Z (S).
- Exit: любая ошибка раскладки чинится одной клавишей за один шаг.

### EPIC 4 — Проактивная установка раскладки + контекст
- **E4.1** ContextProvider: AX-наблюдатель фокуса приложения/поля (M).
- **E4.2** Эвристики роли поля (url/email/search/пароль) → латиница; комбинированный детект защищённого поля, REL-6 (M).
- **E4.3** Память раскладки по приложению/полю + восстановление при фокусе (M).
- Exit: раскладка чаще верна сразу; в полях паролей агент бездействует.

### EPIC 5 — UI и настройки
- **E5.1** Menu bar agent (NSStatusItem, LSUIElement) + быстрый тумблер/индикатор (M).
- **E5.2** Caret-индикатор: borderless non-activating overlay у курсора + fallback в меню-баре (L).
- **E5.3** Settings (SwiftUI): общие / переключение (порог) / хоткеи / исключения + whitelist / словарь / обратная связь; сброс к умолчаниям (L).
- **E5.4** Shadow-mode + панель обзора недавних конвертаций с «в исключения» / «в белый список» (M).
- **E5.5** Обратная связь: звук/вспышка, по умолчанию выкл. (S).
- Exit: всё настраивается из UI; shadow-mode пригоден для калибровки.

### EPIC 6 — Обучение, персистентность, жизненный цикл
- **E6.1** Store исключений/whitelist/памяти/ревертов; атомарная запись; import/export (M).
- **E6.2** Обучение на ручных откатах (интеграция E1/E3) (M).
- **E6.3** Импорт личного корпуса для стартового словаря (S).
- **E6.4** LaunchAgent KeepAlive, single-instance, онбординг разрешений, детект отзыва прав (M).
- **E6.5** Восстановление после сна/смены монитора/смены пользователя, REL-7 (M).
- **E6.6** Перф-проход: CPU≈0% idle, память<60МБ, латентность (S).
- Exit: разделы «Надёжность / Производительность / Приватность» выполнены и проверены.

### EPIC 7 (опционально) — Текстовые инструменты
- Транслитерация, смена регистра, исправление Caps Lock, сниппеты (по M, по приоритету).

## 6. Сквозные требования → где реализуются

- REL-1/REL-2 → InputCapture (+ async). REL-3 → все модули. REL-4 → InputCapture + ContextProvider. REL-5 → LayoutController. REL-6 → ContextProvider. REL-7 → Lifecycle. REL-8 → Lifecycle + Store. REL-9 → TestHarness.
- Производительность (NFR) → событийный InputCapture, словари в память, отсутствие опроса.
- Приватность (SEC) → эфемерный KeystrokeBuffer, локальный Store, отсутствие сетевого кода во всём проекте.

## 7. Реестр рисков и митигации

- **Tap тихо отключается системой** (высокий) → REL-1 перевзведение + watchdog.
- **AXValue-замена не работает в Electron/вебе** (высокий) → надёжный backspace-fallback + отмена при гонке (S0.2 определяет границы).
- **Гонка backspace при быстром наборе** (средне-высокий) → атомарная операция, отмена при смене поля.
- **Двойной модификатор «съедает» обычный Shift** (средний) → временное окно + отмена при другой клавише.
- **Позиция курсора недоступна** для оверлея (средний) → fallback на индикатор в меню-баре.
- **Отзыв разрешений / трение онбординга** (средний) → детект и инструкция.
- **App Sandbox несовместим с глобальным tap** (известный) → Hardened Runtime, без sandbox.

## 8. Стратегия тестирования

- **Unit:** DetectionEngine на корпусе (гейты accuracy/false-positive), round-trip маппинга символов.
- **Integration:** сквозной прогон на матрице приложений (Safari, Chrome, VS Code, Slack, Notes, Terminal) — чек-лист + где можно скрипты.
- **Надёжность:** принудительное отключение tap, нагрузочный тест латентности при быстром наборе, сон/пробуждение, отзыв разрешений.
- **Производительность:** Activity Monitor (idle/active), Instruments по латентности.
- **Приватность:** сетевой монитор (нулевой egress) в суточном прогоне.

## 9. Критический путь

S0.1 → EPIC 2 (пайплайн) с заложенным REL-1/REL-3 — это спина проекта. EPIC 1 идёт параллельно и питает EPIC 2. До UI и проактивности доходим после того, как сквозная автоконвертация стабильна и tap самовосстанавливается. MVP первого запуска: EPIC 1 + EPIC 2 + EPIC 3 + индикатор из EPIC 5 + надёжность из EPIC 6.4–6.5.
