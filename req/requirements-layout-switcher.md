# Требования к локальному переключателю раскладки (личный инструмент для macOS)

## 1. Контекст и цель

Я набираю смешанный текст RU/EN (английский — преимущественно термины внутри русских предложений) и до начала ввода не вижу активную раскладку, из-за чего часто печатаю не в той раскладке и переписываю. Цель — собственное приложение для macOS, которое запускается и работает локально, ничего не передаёт в сеть и помогает всегда печатать в правильной раскладке. Это личный инструмент для одного пользователя, без корпоративного развёртывания и согласований.

## 2. Ключевая проблема и принцип работы

1. До начала ввода неизвестна активная раскладка → текст набирается не в той раскладке.
2. Нужна надёжная автоконвертация ошибочно набранного слова и переключение раскладки, плюс ручная конвертация и ручное переключение.
3. Смешанный RU/EN-ввод не должен ломаться: латинские термины внутри русского текста не конвертируются ошибочно.

**Принцип проектирования.** Стопроцентный автодетект раскладки невозможен (одна и та же последовательность бывает валидна в обеих раскладках). Поэтому цель смещается с «идеального угадывания» на «идеальное ощущение»: правильная раскладка ставится по возможности *до* ввода; автоконвертация срабатывает только при высокой уверенности; при сомнении текст не изменяется; любая остаточная ошибка исправляется мгновенно одной клавишей. Ложная конвертация портит уже правильный текст и раздражает сильнее пропуска — поэтому при неуверенности система бездействует, а ручное исправление делается мгновенным.

Известные сбои аналогов, которых нужно избежать: автопереключение на середине слова («екгst» вместо «trust»), пропуск/нераспознавание терминов (например, fetch), а также тихое «отваливание» перехватчика ввода.

## 3. Функциональные требования

### 3.1 Индикация раскладки
- **FR-1.** Постоянный индикатор текущей раскладки рядом с курсором (caret), обновляется при смене фокуса поля и при смене раскладки.
- **FR-2.** Индикатор в строке меню, опционально — короткое всплывающее уведомление при переключении.
- **FR-3.** Индикацию можно отключить (каждый вариант независимо).

### 3.2 Проактивная установка раскладки (до ввода)
- **FR-4.** Запоминание раскладки по приложению и по типу поля; восстановление при возврате фокуса.
- **FR-5.** Эвристика по роли поля: url, email, поле пароля, поле поиска → латиница.
- **FR-6.** Приоритет проактивной установки над постфактум-конвертацией: цель — чтобы раскладка чаще была верной сразу, а не чинилась после.

### 3.3 Автоматическое переключение
- **FR-7.** Статистический анализ набираемого слова по частотным словарям RU/EN; конвертация фрагмента и переключение раскладки при высокой уверенности.
- **FR-8.** Конвертация строго по границе слова (пробел, Enter, пунктуация). Запрет конвертации на середине слова.
- **FR-9.** Контекстный анализ: учёт языка окружающего/редактируемого текста для снижения ложных переключений.
- **FR-10.** Настраиваемый порог уверенности; ниже порога — бездействие (текст не изменяется).

### 3.4 Петля исправления и ручное управление
- **FR-11.** Одна клавиша: конвертировать последнее слово + переключить раскладку; повторное нажатие откатывает/циклирует результат.
- **FR-12.** Гранулярность операции: слово / строка / выделение.
- **FR-13.** Ручное переключение раскладки одиночным или двойным нажатием модификатора (например, двойной Shift).
- **FR-14.** Hold-to-suppress: удержание модификатора временно глушит автоконвертацию (пароль в обычном поле, код, лицензионный ключ).
- **FR-15.** Отмена последней автоконвертации одной клавишей; не конфликтует с нативным Cmd+Z приложения.

### 3.5 Смешанный ввод RU/EN
- **FR-16.** Пользовательский словарь исключений — слова, которые не конвертируются.
- **FR-17.** Белый список «всегда латиница» под мои термины (API, sprint, backlog, PR и т. п.).
- **FR-18.** Обучение на ручных откатах: повторно не конвертировать слова, которые я вернул вручную.
- **FR-19.** Аккуратная обработка латинских терминов, брендов и аббревиатур внутри русского текста без ложной конвертации.

### 3.6 Самообучение и калибровка
- **FR-20.** Shadow-mode: режим «только показывать/логировать локально, что было бы сконвертировано», без изменения текста — для безопасной калибровки порога перед включением реальной конвертации.
- **FR-21.** Локальный обзор недавних автоконвертаций с кнопками «в исключения» и «в белый список» — для самотюнинга со временем.
- **FR-22.** Импорт личного текста для стартового наполнения словаря и оценки точности.

### 3.7 Дополнительные функции по работе с текстом (опционально, по приоритету)
- **FR-23.** Транслитерация выделенного текста (кириллица ↔ латиница).
- **FR-24.** Смена регистра выделенного текста (UPPER / lower / Каждое Слово).
- **FR-25.** Исправление случайного Caps Lock и двух заглавных в начале слова.
- **FR-26.** Автозамена / текстовые сниппеты (короткое сокращение → длинная фраза), локальный список.
- **FR-27.** Явно ВНЕ функционала: никакого «дневника»/логирования набранного текста.

### 3.8 Фоновая работа и поведение приложения
- **FR-28.** Фоновый агент в строке меню: без иконки в Dock, без окна (LSUIElement).
- **FR-29.** Автозапуск при входе в систему (LaunchAgent), быстрый холодный старт.
- **FR-30.** Глобальное включение/выключение автоконвертации из меню и горячей клавишей.
- **FR-31.** Автоотключение в полноэкранных приложениях/играх. Основной механизм — детект полноэкранного режима с захватом ввода; ручной список приложений — дополнение, а не основа.
- **FR-32.** Чёрный список приложений и типов полей, где агент неактивен (по умолчанию — поля паролей).

### 3.9 Меню и тонкая настройка
- **FR-33.** Меню в строке меню: текущая раскладка, быстрый тумблер автопереключения, shadow-mode, доступ к настройкам, выход.
- **FR-34.** Окно настроек по разделам: общие, переключение (порог, поведение на границе слова), горячие клавиши (одиночный/двойной модификатор), исключения и белый список, словарь (импорт/экспорт локально), обратная связь (звук/вспышка, по умолчанию выкл.).
- **FR-35.** Сброс к настройкам по умолчанию.

## 4. Производительность и ресурсы

- **NFR-1.** Событийная архитектура (CGEventTap / глобальный монитор) без опроса в цикле — нагрузка только в момент ввода.
- **NFR-2.** CPU в простое ≈ 0%; при активном вводе — единицы процентов.
- **NFR-3.** Память в покое — менее ~60 МБ.
- **NFR-4.** Добавляемая задержка на нажатие — в пределах ~5–10 мс, без потери и дублирования символов даже при быстрых сериях.
- **NFR-5.** Словари загружаются в память один раз, компактные; отсутствует фоновый дисковый ввод-вывод в покое.
- **NFR-6.** Незначимое влияние на батарею: нет таймеров пробуждения и удержания системы от сна.
- **NFR-7.** Нативная реализация на Swift, минимум зависимостей; никакого фонового сетевого стека.

## 5. Надёжность и отказоустойчивость

Это причины, по которым аналоги «вдруг перестают работать»; раздел обязателен для стабильного результата.

- **REL-1.** Авто-перевзведение event tap: macOS отключает CGEventTap при таймауте колбэка или высокой нагрузке — это событие нужно ловить и немедленно включать tap заново.
- **REL-2.** Неблокирующий колбэк: горячий путь < 1–2 мс, вся тяжёлая логика выполняется асинхронно. Медленный колбэк = система убивает tap и появляется лаг ввода.
- **REL-3.** Fail-open: любая ошибка или неопределённость → символ проходит без изменений. Никогда не «съедать» и не портить ввод.
- **REL-4.** Безопасность композиции/IME: не вмешиваться во время ввода через метод, dead keys, диктовку и accent-меню.
- **REL-5.** Замена текста через Accessibility (AXValue) там, где приложение это поддерживает; иначе — синтетические Backspace. При гонке или изменении поля во время операции — отмена операции.
- **REL-6.** Комбинированный детект защищённых полей: EnableSecureEventInput + роль AXSecureTextField + чёрный список доменов (веб-поля паролей часто не дают secure input).
- **REL-7.** Восстановление состояния после сна, смены монитора и смены пользователя.
- **REL-8.** Один инстанс + KeepAlive в LaunchAgent (всегда запущен, авто-рестарт после краха); атомарная запись словаря и настроек.
- **REL-9.** Детерминизм поведения и регрессионный тест на личном корпусе при каждом изменении; точность и доля ложных конвертаций — контролируемые метрики.

## 6. Приватность и локальность

- **SEC-1.** Нулевой исходящий сетевой трафик: никакой телеметрии, аналитики, облака и сетевых проверок обновлений. Полностью офлайн.
- **SEC-2.** Никакого постоянного хранения набранного текста и «сырых» нажатий; в памяти — только минимум для анализа текущего слова, буфер очищается сразу.
- **SEC-3.** Локально хранятся только пользовательский словарь и настройки, под контролем пользователя.
- **SEC-4.** Уважение Secure Input (EnableSecureEventInput): полное бездействие в защищённых полях.
- **SEC-5.** Запрашиваются только необходимые разрешения: Input Monitoring / Accessibility.

## 7. Совместимость и технические ограничения

- Перехват ввода: CGEventTap / глобальный NSEvent-монитор; нужны разрешения Accessibility + Input Monitoring (выдаются вручную один раз).
- Совместимость: текущая и предыдущая мажорная версия macOS, Apple Silicon (Intel — опционально).
- Полноценный App Sandbox конфликтует с глобальным перехватом ввода — для личной сборки достаточно Hardened Runtime; распространение не требуется.
- Не конфликтует с системными сочетаниями и нативным переключением раскладки.
- Частотные словари RU/EN поставляются в комплекте и работают локально.

## 8. Вне scope

- Любая сетевая функциональность, синхронизация между устройствами, облако.
- Логирование/дневник набранного текста.
- Другие ОС и языки, кроме RU/EN (немецкий и пр. — при желании позже).
- Проверка орфографии и автодополнение.

## 9. Критерии приёмки

Целевая формулировка — не «100% автодетекта», а ноль порчи правильного текста и мгновенное восстановление.

- Ноль порчи уже правильно набранного текста за сутки реального использования (главный критерий).
- Доля ложных конвертаций (порча верного текста) ≤ 0.5%; пропуски допустимы, ложняки — нет.
- Любая остаточная ошибка раскладки исправляется одной клавишей за один шаг.
- Event tap самовосстанавливается после принудительного отключения и под нагрузкой (тест).
- Конвертация только по границе слова (тест на словах вроде «trust», «fetch»).
- Нет вмешательства в поля паролей и в IME-ввод (тест).
- CPU в простое ≈ 0%, память в пределах NFR-3 (Activity Monitor).
- Подтверждённый нулевой сетевой трафик при суточном прогоне.
- Задержка ввода в пределах NFR-4 на быстром наборе.
